on COLLABORATION

How does the saying go?

"If you want to go fast, go alone. If you want to go far, go together."

It's going to be important have a workflow for Tastemakers Africa's engineering/software team that supports scalability, growth, and collaboration. The same tools I used to build this site, Vagrant, Chef, and Github are excellent for team collaboration.

I'll bring to Tastemakers Africa a workflow, using those tools and others, that will address the common challenges of building products across distributed teams like tedious environment setup, discrepancies between development and production environments, and clashes in code written by different developers.

The Teamwork Formula

Vagrant
$ vagrant up

It's that simple. Typing those two words into the command line will give every developer on the team the same setup to develop on.

Why is that important? There is a strong possibility if everyone is not working on the same setup that code may work on someone's laptop but not on a live server. Troubleshooting problems like that can cost time and money.

Chef

Chef works hand-in-hand with Vagrant to automate installation of software on servers. I wrote Chef "cookbooks" for this site that automatically install nginx, nodejs, and other supporting software and configurations.

Why is that important? In order to grow Tastemakers Africa is going to need an infrastructure that goes beyond a routine setup that is useful for something like WordPress. Again a time and money argument can be made for a team automating with Chef. Time is better spent developing applications for customers than setting up operating systems.

Git

Collaboration on code, while it can be powerful if done right it can get messy. Git and Github are tools that can help work across teams and avoid conflicts.

Why is that important? Those conflicts can again cost time and money and Git keeps track of your changes in several locations; we've all lost a flash drive.

#branch off of the "develop" branch to create a new feature
$ git checkout -b newfeature develop
#write code for the new feature on a different branch locally (remember Vagrant and Chef)
#switch to the development branch
$ git checkout develop
#merge your new feature into the development branch
$ git merge --no-ff newfeature

Using this method to code on a team keeps track of all the changes made to the code and "quarantines" experimentation to create new features.
